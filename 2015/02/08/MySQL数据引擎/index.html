MySQL数据引擎笔记

sql 2015-02-08

MySQL数据引擎分类

  1. ISAM

    优点：ISAM执行读取操作的速度很快，而且不占用大量的内存和存储资源。

    缺点：不支持事务处理，也不能够容错。 如果你的硬盘坏了，那么数据文件就无法恢复了。

  2. MyISAM（MySQL的ISAM扩展格式和缺省的数据库引擎，MySQL 5.0 之前的默认数据库引擎）

    优点：MyISAM使用了一种表格锁定的机制，来优化多个并发的读写操作，其代价是你需要经常运行OPTIMIZE TABLE命令，来恢复被更新机制所浪费的空间。MyISAM强调了快速读取操作。

    缺点：不能在表损坏后恢复数据。

  3. HEAP

    优点：HEAP允许只驻留在内存里的临时表格。驻留在内存里让HEAP要比ISAM和MYISAM都快。

    缺点：管理的数据是不稳定的，而且如果在关机之前没有进行保存，那么所有的数据都会丢失。

  4. InnoDB（MySQL 5.5 起成为默认数据库引擎）

    InnoDB包括了对事务处理和外来键的支持，这两点都是前两个引擎所没有的。

项目中常用的两种引擎

在实际项目中最常用的就是MyISAM和InnoDB，通常来说这两种引擎的应用场景如下：

一般来说，MyISAM适合：

  1. 做很多count 的计算；
  2. 插入不频繁，查询非常频繁；
  3. 没有事务。

InnoDB适合：

  1. 可靠性要求比较高，或者要求事务；
  2. 表更新和查询都相当的频繁，并且表锁定的机会比较大的情况。
