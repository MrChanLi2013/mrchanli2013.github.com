MySQL优化方案笔记

sql 2015-03-08

一些常用的优化手段

  1. 选择优化的数据类型

    • 更小通常便好

      一般来说，要试着使用能正确的储存和表示的数据类型。因为他们使用了更少的磁盘空间。

    • 简单就好

      越简单的数据类型，需要的CPU周期就越少。例如：应该使用数据库时间类型保存时间而不要使用字符串；应该使用整数来保存IP地址， IP地址本质就是无符号的32位整数，之所以用添加小数点是为了可读性。 MySql中提供了INET_ATON()和INET_NTOA()。

    • 尽量避免使用NULL

  1. 使用高性能的索引

    • 如果在查询没有隔离索引的列，MySQL通常不会使用索引。隔离列意味着它不是表达式的一部分，也没有位于函数中。

      下面的查询不会使用p_id上的索引，p_id属于了表达式的一部分：

       mysql> select p_id from table where p_id + 1 = 4;

      下面也是一种常见的错误：

       mysql> select ... from table where TO_DAYS('2015-09-09')-TO_DAYS(date_col) <= 10;

      这个查询会查找date_col值离今天不超过10天的所有行，但它不会使用索引，因为使用了TO_DAYS函数。可以优化为：

       mysql> select ... from table where date_col >= DATE_SUB('2015-09-09',INTERVAL 10 DAY)
