Html语义化笔记

html 2015-06-18

  1. 什么是html语义？

    直观的说，就是html的含义，从html代码中使用的标签本身就可以说明标签包含内容的含义，比如使用<h1>标签，则代表所包含的内容是标题。

  2. 如何做到高语义化的html？

    要编写高语义化的html代码需要足够的耐心和良好的编码意识，具备了最基础的要求之后就需要在实际的项目中不断的实践了。当然也是有一些公认的比较好的实践原则：

    • 熟悉所有规范中的html标签，理解个标签的语义，在合适的地方使用合适的标签。

      例如：<hx>系列标签则表示标题，在网页中展示各层级的标签时使用；<ul>，<ol>表示列表，在展现各种数据列表或者菜单时使用；其他使用频率较高的有<p>，<em>，<strong>，<table>，<site>，<blockquote>等。此外还有一些html5新加入的标签，包括：<header>，<footer>，<article>，<section>，<nav>，<aside>。这些在W3C规范中针对所有标签都有详细的解释和使用场景示例。

    • 熟悉各个标签上规范的属性，给html标签设置必要的属性

      在很多前端规范中都会提到两个属性是alt和title。这两个属性的设置同样也提高了Html的语义。在<img>中alt是必须要设置的属性。

       <!-- alt属性除了对图片做解释说明之外，也作为图片在浏览器中未加载时显示的代替文字 -->
 <img src="/img/loading.gif" alt="data loading"/>

 <!-- title 属性是对元素的解释说明，并且作为浏览器中当鼠标移动到元素上时显示的提示文本 -->  
 <a href="" title=""></a>

    • 样式和结构分离

      首先要将单纯的css样式代码和html代码分离。更进一步，就是要把html代码中用于表达外观的部分从html中删除，利用css实现。简洁的html代码会让语义更加明确。

       <ul><li>|</li></ul>

      在上面的<li>中有一个分隔符，其实它不属于网页的内容，应该从html中去掉。在css中提供了:before，:after两个伪类，很适合这样的场景。

       <div>
   <div class="sidebar">this is a side bar.</div>
   <div class="content">this is a the main content.</div>
   <div class="clear"></div>
 </div>

 //对应的css

 .sidebar{float:left;width:150px}
 .content{float:right;width:450px}
 .clear{clear:both}

      为了清除元素的浮动，上面代码添加了一个空的<div>，很多人认为这是很正常的做法，其实有更好的解决方案：

       <div class="clearfix">
   <div class="sidebar">this is a side bar.</div>
   <div class="content">this is a the main content.</div>
 </div>

 //相应的css类clearfix的声明为：

 .clearfix {
   *zoom:1;
 }
 .clearfix:before,.clearfix:after{
   display:table;
   content:"";
 }
 .clearfix:after{
   clear:both;
 }

      还有<br/>，很多喜欢有来加大元素直接的行距，这个就是完全没有理解到该元素的语义，该标签仅仅用于文本内容中的换行。如果要实现元素之间的换行或者增加间距，可以通过设置display，margin，padding来实现。

      还有就是<img>的使用，什么时候使用背景图，什么使用<img>，其原则就是如果图片仅仅起到装饰作用，则使用背景图，如果是属于页面的内容则应该使用<img>。

    • 给空标签中添加隐藏文字，用于说明标签的实际功能

      这个一般式使用text-indent来达到隐藏文字的目的。
