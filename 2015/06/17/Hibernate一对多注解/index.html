Hibernate一对多注解笔记

java 2015-06-17

Hibernate中一对多注解@OneToMany

parent和child直接就属于一对多关系，一个parent可能有多个child，一个child只有一个parent：

Parent类：

@Entity
@Table(name="parent")
public class Parent {
  @Id
  @GeneratedValue
  private Integer id;

  @OneToMany(cascade = CascadeType.ALL)
  private Set<Child> children;

  @Column
  private String name;

  //getter and setter
}

Child类：

@Entity
@Table(name="child")
public class Child {
  @Id
  @GeneratedValue
  private Integer id;

  @Column
  private String name;

  //getter and setter
}

在Parent配置@OneToMany，这时Hibernate会创建第3张关联表，parent_child[parent_id,child_id]。

Test：

Parent parent = new Parent();
parent.setName("张三");
Set<Child> childSet = new HashSet<Child>();
childSet.add(new Child("小明"));
childSet.add(new Child("小王"));
parent.setChildren(childSet);
//session：Hibernate session
session.save(parent);

数据库中parent中会存储一条记录，child会存储两条记录，其中关联表存储结果如下：

parent_id child_id
1 1
1 2

如果不想Hibernate创建关联表：

  1. 更改Person如下，在children上添加注解@JoinColumn(name=”parent_id”)，那么在child表中hibernate会自动创建新的一列parent_id，并且此parent_id为外键。

     @OneToMany(cascade = CascadeType.ALL)
 @JoinColumn(name = "parent_id")
 private Set<Child> children;

  2. 也可以给Person的@OneToMany添加（mappedBy = “parent”），在Child中添加@ManyToOne：

     //Person mappedBy指定关系的维护方为Child
 @OneToMany(cascade = CascadeType.ALL, mappedBy = "parent")
 private Set<Child> children;

 //Child
 @ManyToOne
 private Parent parent;

 //Test
 Parent parent = new Parent();
 parent.setName("张三");
 Set<Child> childSet = new HashSet<Child>();
 Child child1 = new Child("小明");
 //如果此处child不设置parent，则在数据库中parent_id字段就为空，因为使用mappedBy
 child1.setParent(parent);
 Child child2 = new Child("小王");
 //
 child2.setParent(parent);
 childSet.add(child1);
 childSet.add(child2);
 parent.setChildren(childSet);
 session.save(parent);
