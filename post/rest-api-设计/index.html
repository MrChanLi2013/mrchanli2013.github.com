验证

使用Basic authentication

如果没有验证就访问接口返回HTTP状态码403 Forbidden。但如果我们有接口是私有的我们可以返回404 Not Found，这样如果该地址被暴露出去之后，别人访问的时候会认为这个地址是错误的。也或者说我们只是针对部分用户，比如A、B两个用户A有权限访问接口，B没有权限，那么B访问的时候我们就返回404，A访问的时候没有验证成功我们就返回403。

登录失败的情况
  • 无效的凭据：401 Unauthorized
HTTP/1.1 401 Unauthorized
{
	  "message": "用户名或密码错误"
}
  • 凭据输入错误次数过多：403 Forbidden
 HTTP/1.1 403 Forbidden
 {
	 "message": "操作过于频繁，请稍后操作"
 }

参数

GET参数

使用查询的方式（HTTP query string parameter），http://api.com/product/1?status=normal，获取id为1并且状态为normal的产品

  • 其中status=normal就使用查询参数的方式
  • id为1可以使用{id}方式存在path中：http://api.com/product/{id}?status=normal
  • 如果是中文的查询参数，建议使用encodeURIComponent()或者encodeURI()将url encode
POST、PATCH、PUT、DELETE参数

这些方法的参数使用json的格式传输，Content-Type = 'application/json'

客户端错误定义

1、发送的参数json格式错误：400 Bad Request
HTTP/1.1 400 Bad Request

{"message":"json解析失败"}
2、参数验证错误：422 Unprocessable Entity
HTTP/1.1 422 Unprocessable Entity

{
  "message": "参数验证失败",
  "errors": [
    {
      "message": "title不能为空",
      "field": "title",
      "code": "missing_filed"
    }
  ]
}

code可能存在几种：

  • missing：资源不存在
  • missing_filed：实体的字段缺失
  • invalid：实体的字段格式验证失败
  • already_exists：实体已存在

分页

当请求返回会有多条时，采用分页形式，可以设置默认一页显示30，可以使用?page参数查询第几页，?per_page参数设置每页返回多少条，可以使用?sort进行排序。

HTTP 响应状态码

2XX（请求成功）
  • 200 Ok：POST、GET等方法请求成功返回
  • 201 Created：创建某个资源成功时返回
  • 204 No Content：请求成功了，没有返回任何东西，在DELETE方法中最常见
4XX（客户端错误）

可以参看之前的介绍

  • 400 Bad Request
  • 401 Unauthorized
  • 403 Forbidden
  • 404 Not Found
5XX（服务器错误）
  • 500 Internal Server Error：服务器错误
