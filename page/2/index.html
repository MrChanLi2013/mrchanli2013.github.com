使用Basic authentication

如果没有验证就访问接口返回HTTP状态码403 Forbidden。但如果我们有接口是私有的我们可以返回404 Not Found，这样如果该地址被暴露出去之后，别人访问的时候会认为这个地址是错误的。也或者说我们只是针对部分用户，比如A、B两个用户A有权限访问接口，B没有权限，那么B访问的时候我们就返回404，A访问的时候没有验证成功我们就返回403。

登录失败的情况
  • 无效的凭据：401 Unauthorized
HTTP/1.1 401 Unauthorized
{
	  "message": "用户名或密码错误"
}
  • 凭据输入错误次数过多：403 Forbidden
 HTTP/1.1 403 Forbidden
 {
	 "message": "操作过于频繁，请稍后操作"
 }

Vue.js最佳实践

2018-01-22
前端
约 313 字 预计阅读 1 分钟

@[web,javascript]

1. Vue中是否还需要使用JQuery？

前端开发相信对jQuery非常熟悉了，对于刚开始使用vue的人来说，会更倾向于直接使用jQuery操作DOM，更加方便快捷。不过现在是时候改变一下你的习惯了，可以不再使用jQuery。

Vue也提供了类似的API：

<template>
  <div>
    ...
    // 作用关于dom元素上
    <p ref="myElement">This is the content of my element.</p>
    <my-component ref="component">This is custom component.</my-component>
    ...
  </div>
</template>
<script>
	export default {
	  mounted () {
	    console.log(this.$refs.myElement)
	    console.log(this.$refs.component)
	  }
	}
</script>
