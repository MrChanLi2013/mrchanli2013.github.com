Web分布式session管理

Web 2016-07-25

摘要

随着web功能的强大起来，用户数量也多起来，这就需要在应用服务器前增加负载均衡设备来解决，例如有两台设备，当用户访问的时候是随机登录的一台设备，那这时候session如何处理？

解决方案

  1. Session Stick
    nginx支持stick技术。

  2. Session复制
    将session复制至没一台服务器上。

  3. Session集中管理
    使用单独的session服务器，将用户访问先路由至session服务器。

  4. 基于Cookie管理
    将session信息放在cookie中，再次访问服务器时，将cookie信息带上发送至服务器，这样就可以验证身份信息了。
    缺点：

    • cookie信息不安全
    • 每次都需要携带cookie信息，增加了网络开销
    • cookie长度有限制
