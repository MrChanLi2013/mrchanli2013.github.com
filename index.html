了解微服务架构

2018-04-10
软件架构
约 1843 字 预计阅读 4 分钟

“微服务”一词源于Martin Fowler 的名为Microservices 的博文， 可以在他的官方博客 上找到： Martin Fowler Blog

更多内容请阅读书籍《SpringCloud微服务实战-翟永超》。

什么是微服务架构

简单地说，微服务是系统架构上的一种设计风格， 它的主旨是将一个原本独立的系统拆分成多个小型服务，这些小型服务都在各自独立的进程中运行，服务之间通过基于HTTP的RESTful API进行通信协作。被拆分成的每一个小型服务都围绕着系统中的某一项或一些耦合度较高的业务功能进行构建， 并且每个服务都维护着自身的数据存储、业务开发、自动化测试案例以及独立部署机制。由于有了轻量级的通信协作基础， 所以这些微服务可以使用不同的语言来编写。

使用Basic authentication

如果没有验证就访问接口返回HTTP状态码403 Forbidden。但如果我们有接口是私有的我们可以返回404 Not Found，这样如果该地址被暴露出去之后，别人访问的时候会认为这个地址是错误的。也或者说我们只是针对部分用户，比如A、B两个用户A有权限访问接口，B没有权限，那么B访问的时候我们就返回404，A访问的时候没有验证成功我们就返回403。

Oracle下常用函数

2017-11-08
数据库
约 558 字 预计阅读 2 分钟

Oracle中经常使用聚合函数，那么除了这些函数，还有哪些我们会经常用到呢？

总结如下：

  • round()：四舍五入方法，SELECT ROUND( number, [ decimal_places ] ) FROM DUAL，decimal_places 数值是按照指定的小数位进行四舍五入，小数位是从0开始计算与数组下标一样
 select round(45.55, 1) from dual; -- 输出45.60
 select round(45.21, 1) from dual; -- 输出45.20
  • to_char()转换日期
 select to_char(sysdate 'YYYY-MM-DD') from dual --转换成年-月-日的格式
  • to_char()保留两位小数补0，例如：在java中查询出的字段映射到java class类属性时需要要补0
 TO_CHAR(35.00, 'FM9999999990.00')
